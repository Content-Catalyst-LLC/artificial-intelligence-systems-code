# Model Card Notes for NLP and Computational Language Systems

Recommended documentation fields:

- model family
- tokenizer
- context window
- training data sources
- language and domain coverage
- retrieval system details
- intended use
- out-of-scope use
- known hallucination risks
- citation and grounding behavior
- domain-specific limitations
- prompt sensitivity
- privacy and retention rules
- human oversight process
- correction and appeal mechanisms
