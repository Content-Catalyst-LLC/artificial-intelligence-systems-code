# Explanation Audit Checklist

## Explanation Purpose

- Intended explanation audience is defined.
- Explanation purpose is defined: debugging, user support, audit, contestability, or governance.
- Explanation method is appropriate for the model and use case.

## Fidelity and Stability

- Explanation fidelity has been tested.
- Explanation stability has been tested across similar cases.
- Explanation parameters and background data are documented.

## Counterfactual Review

- Counterfactuals are valid.
- Counterfactuals are plausible.
- Counterfactuals avoid protected or inappropriate feature changes.
- Counterfactuals support meaningful recourse where applicable.

## Causal Review

- Associational explanations are not presented as causal claims.
- Causal review is required for intervention-oriented decisions.
- Confounding and proxy variables are documented.

## Governance

- Explanation artifacts are logged.
- Model version and data version are linked.
- Reviewer decisions are recorded.
- Contestation process is documented.
