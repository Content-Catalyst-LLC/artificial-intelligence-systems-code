# Model Validation Card Notes

Recommended documentation fields:

- model name
- model version
- intended use
- prohibited uses
- training data description
- validation data description
- test data description
- split design
- leakage checks
- metric rationale
- performance summary
- subgroup performance
- calibration results
- uncertainty results
- distribution-shift tests
- external validation
- benchmark limitations
- known failure modes
- monitoring plan
- governance owner
