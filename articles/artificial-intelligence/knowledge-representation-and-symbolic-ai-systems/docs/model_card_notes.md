# Documentation Notes for Knowledge Representation and Symbolic AI Systems

Recommended documentation fields:

- knowledge base purpose
- domain scope
- representation type
- ontology or schema version
- class hierarchy
- relation vocabulary
- rule inventory
- default and exception handling
- inference engine
- provenance sources
- governance owner
- rule review cycle
- inference trace format
- known limitations
- unresolved ambiguity
- human review triggers
- deprecation policy
