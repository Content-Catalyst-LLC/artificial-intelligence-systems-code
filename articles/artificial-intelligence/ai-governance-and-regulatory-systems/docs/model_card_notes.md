# Governance Documentation Notes

Recommended documentation fields:

- system purpose
- intended use
- prohibited-use screening
- affected population
- risk tier
- data provenance
- model version
- vendor or internal owner
- evaluation evidence
- bias and fairness testing
- robustness and security testing
- human oversight design
- transparency and notice process
- appeal or contestation process
- incident response plan
- post-deployment monitoring plan
- residual risk acceptance
- review and approval history
