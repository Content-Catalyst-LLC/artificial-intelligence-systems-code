# AI, Information Integrity, and Media Systems

This repository folder supports the article:

**AI, Information Integrity, and Media Systems**

It provides reproducible examples for information-integrity risk scoring, source-diversity monitoring, provenance coverage, correction-effectiveness analysis, synthetic-media review, platform-governance documentation, media-system monitoring, and public-accountability workflows.

## Purpose

AI systems now shape how information is created, summarized, translated, ranked, recommended, verified, monetized, corrected, and trusted. This repository treats information integrity as a media-system governance problem rather than a narrow content-moderation problem.

The examples here examine:

- how to score information-integrity risk;
- how provenance gaps and amplification risk affect governance priority;
- how source diversity can be monitored;
- how correction effectiveness can be estimated;
- how synthetic media and AI-assisted publication records can be documented;
- how platform governance actions can be tracked;
- how media-system indicators can support public accountability;
- how information integrity can be evaluated without collapsing into censorship or centralized content control.

## Contents

- `python/information_integrity_risk_scoring.py` — content-level information-integrity risk scoring
- `python/requirements.txt` — Python dependencies
- `r/source_diversity_media_monitoring.R` — source diversity, provenance coverage, correction effectiveness, and low-integrity amplification workflow
- `sql/media_integrity_governance_schema.sql` — governance schema for provenance, sources, corrections, amplification, moderation actions, and public reporting
- `rust/` — command-line information-risk checker
- `go/` — lightweight media-integrity monitoring service
- `julia/source_diversity_sensitivity.jl` — source-diversity sensitivity analysis
- `typescript/` — media-integrity governance record validator
- `cpp/` — high-throughput information-risk scoring example
- `docs/templates/` — provenance review, editorial AI disclosure, correction tracking, synthetic-media review, platform-governance, public-reporting, and audit templates
- `notebooks/` — notebook placeholder for expanded analysis
- `scripts/run_all.sh` — runs available workflows and smoke checks
- `outputs/` — generated workflow outputs
