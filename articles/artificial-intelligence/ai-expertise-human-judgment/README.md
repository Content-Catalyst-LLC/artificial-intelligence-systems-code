# AI, Expertise, and Human Judgment

This repository folder supports the article:

**AI, Expertise, and Human Judgment**

It provides reproducible examples for human-AI decision simulation, expert disagreement analysis, automation-bias monitoring, reviewer calibration, governance documentation, skill-preservation checks, and judgment-centered AI workflows.

## Purpose

AI systems increasingly support expert work in medicine, law, engineering, science, education, finance, public administration, sustainability, infrastructure, and organizational strategy. This repository treats AI-assisted expertise as a sociotechnical system rather than a simple automation problem.

The examples here examine:

- when AI support improves human judgment;
- when AI reliance becomes excessive;
- how expert disagreement can be monitored;
- how contextual complexity should affect AI reliance;
- how skill atrophy and deskilling risks can be documented;
- how institutions can preserve reviewability, accountability, and professional responsibility.

## Contents

- `python/human_ai_decision_support.py` — human-AI decision simulation with reliance, disagreement, and review flags
- `python/requirements.txt` — Python dependencies
- `r/expert_disagreement_monitoring.R` — expert disagreement and review-quality analysis
- `sql/expertise_governance_schema.sql` — governance schema for AI-supported expert review
- `rust/` — command-line reliance threshold checker
- `go/` — lightweight expert-review monitoring service scaffold
- `julia/reliance_sensitivity.jl` — AI reliance sensitivity analysis
- `typescript/` — governance record validator
- `cpp/` — high-throughput reliance and disagreement scoring
- `docs/templates/` — expert review, disagreement, audit, training, and skill-monitoring templates
- `notebooks/` — notebook placeholder for expanded analysis
- `scripts/run_all.sh` — runs available workflows and smoke checks
- `outputs/` — generated workflow outputs
